# PATCHNOTES — Roguelike CardGame

История изменений проекта. **Новые записи — сверху.**

Формат записи — по сессиям/датам, с группировкой:
**✨ Добавлено · 🔧 Изменено · 🐛 Исправлено · 🗑 Удалено · ⚙️ Инфраструктура**

## Сессия 29 — 4 июня 2026

### 🔧 Изменено (рефакторинг — Stage 3: остальной UI)
- **`ui/Shop.py` (273) → пакет `ui/shop/`.** `data` (пул карт + цены + генерация витрины),
  `main_view` (витрина + кнопки), `remove_view` (утилизация колоды), `base` (состояние-машина
  MAIN/REMOVE + клики). Импорт `from ui.shop import Shop`.
- **`ui/GameView.py` (271 → 140).** `HoverState` → `ui/hover_state.py`; `DRAW_HANDLERS` + обёртки
  экранов → `ui/draw_dispatchers.py`; боевой hover-расчёт → `ui/combat/hover.py`. Удалён мёртвый
  `_draw_placeholder`. Класс `GameView` остался точкой входа (`from ui.GameView import GameView`).
- **`ui/VictoryScreen.py` (226) → пакет `ui/victory/`.** `data` (палитра), `rewards_view`
  (список наград + кнопки), `modal` (подтверждение), `base` (состояние + оркестрация + клики).
- **`ui/CardLibraryView.py` (167) → пакет `ui/library/`.** Списки карт по классам + вкладки →
  `data.py` (растут с контентом), отрисовка/логика → `base.py`.
- `ui/MapView.py` (167) оставлен как есть: цельный render-метод под мягким потолком 220,
  уже делегирует иконки/геометрию в `map_icons.py` — дробление повредило бы связность.
- **Рефакторинг крупных файлов (Stage 1–3) завершён.** Все файлы логики ≤150; UI — по швам.

### 🔧 Изменено (рефакторинг — Stage 2: крупный UI)
- **`ui/HubView.py` (386) → пакет `ui/hub/`.** Разбит по швам: `data` (палитра + CLASS_INFO +
  геометрия), `selectors` (карточки классов), `deck` (анимированная стопка), `base` (состояние +
  оркестрация + клики). Импорт `from ui.hub import HubView`.
- **`ui/CombatInterface.py` (370) → пакет `ui/combat/`** (рядом с существующим `hud.py`):
  `layout` (палитра+геометрия), `panels` (релик-бар + панели игрока/врага + лог), `bottom`
  (рука/стопки/кнопка хода), `interface` (оркестратор + тултипы). Импорт `from ui.combat import CombatInterface`.
  Попутно удалён мёртвый дубль `CombatInterface.draw_ability_slot` (реальный — в `CombatHUD`).
- **`ui/CardRenderer.py` (357) → пакет `ui/cards/`.** Фасад `CardRenderer` в `renderer.py`,
  внутренности: `classifier` (тип карты), `description` (заголовок + умное описание),
  `keywords` (тултип-глоссарий), `data` (палитра/цвета). Все ~8 импортёров переведены на
  `from ui.cards import CardRenderer`.
- **Введён дифференцированный ГОСТ размера файла:** логика (`core/`,`managers/`) — жёстко 150;
  UI — деление по смыслу (data/rendering/handlers как `ui/chest/`), мягкий потолок рендер-файла ~220.

### 🔧 Изменено (рефакторинг — Stage 1: контент-ядро)
- **`core/players/abilities.py` (248 стр.) → пакет `core/players/abilities/`.** По одному файлу
  на способность (`warrior`/`rogue`/`mage`/`druid`/`berserker`), реэкспорт через `__init__.py`.
  Базовый `ClassAbility` остался в `core/players/ability.py`. Точка входа
  `from core.players.abilities import *Ability` не изменилась.
- **`core/relics/advanced.py` (269 стр.) → пакет `core/relics/advanced/` (по теме).**
  Файлы: `bleed_poison` (кровь/яд), `shield` (щит), `healing` (исцеление), `damage` (урон/дебафф),
  `utility` (экономика). `RELIC_POOL`/`ALL_RELICS` и все импорты неизменны (`ALL_RELICS` = 19).
- **Разобран god-object `GameManager` (266 → 145 стр.).** Спавн врага вынесен в
  `managers/EnemySpawner.py` (`build_enemy` + `ENEMY_REGISTRY` + формулы статов), расчёт наград —
  в `managers/RewardManager.py` (`build_rewards`: золото/реликвия/ключ). Методы
  `spawn_procedural_enemy`/`distribute_combat_rewards` остались тонкими фасадами с прежними
  сигнатурами; публичный интерфейс `gm.*` не изменился. `ENEMY_REGISTRY` реэкспортируется из
  `GameManager` для совместимости.

## Сессия 28 — 3 июня 2026

### ⚙️ Инфраструктура
- Настроен **GitHub CLI** (`gh` 2.93.0): авторизация, git credential helper, дефолтный репозиторий, scope `workflow`.
- Добавлен **CI на GitHub Actions** (`.github/workflows/ci.yml`): линт `ruff` (критичные ошибки) → `compileall` (включая `ui/`, без запуска Pygame) → проверка импортов `core`/`managers` → smoke-тест `BalanceSimulator` (1500 боёв). Запуск на push/PR в `dev` и `main`, Python 3.10. Экшены на Node 24 (`checkout@v6`, `setup-python@v6`).
- Добавлен `requirements.txt` (pygame, pillow, requests).

### 🐛 Исправлено
- **Берсерк «зомби-ход».** «Кровавая ярость» бьёт игрока сквозь щит и могла оставить его на 0 HP до конца хода (смерть ловилась только в `end_turn_phase`). Логика смерти вынесена в `CombatManager.check_player_defeat()` и вызывается также после активации способности (`InputHandler`). Теперь смерть от своей способности сразу ведёт на экран лидеров.
- **Пассив Друида** «Токсичный круговорот» теперь передаёт `combat_manager` в `add_status` (соблюдение соглашения проекта).
- Реликвия **«Гнилой Клык»**: удалён мёртвый флаг `_intercepted` и лишние оверрайды хуков (поведение не изменилось — логика в `Creature.tick_statuses`).
- Удалён недостижимый битый код в `VampireDamageEffect.execute` (`core/cards/base.py`) — обращался к несуществующим `base`/`gm_ref` после `pass`. Найден линтером CI.

### 🗑 Удалено
- 5 мёртвых легаси-файлов из корня `core/` (`Card.py`, `Relic.py`, `enemy.py`, `player.py`, `items.py`) — домодульные остатки, не импортировались нигде. Проверено после удаления: lint, compileall, импорты, BalanceSimulator, спавн врагов — всё работает.

### 📝 Документация
- Проведён аудит зависимостей и боевой логики (см. раздел «Аудит Сессии 28» в `_project_map.md`).
- `_project_map.md` синхронизирован с кодом: исправлена секция «Пассивы классов» (все 3 пассива описаны неверно), добавлены разделы «Игровой цикл и машина состояний», «Карты и эффекты», «Бой: цикл хода», «Колода», «Враги: намерения», «Как добавить контент». Обновлён устаревший workflow (локальная работа вместо выкачки из GitHub).
- Заведён этот файл (`PATCHNOTES.md`).
