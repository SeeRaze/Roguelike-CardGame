# Card Fusion — дизайн-док (v0.1, ЧЕРНОВИК ДЛЯ ОБСУЖДЕНИЯ)

_Статус: НЕ реализовано. Это артефакт для разгона с юзером (§2 дорожной карты ядра
[[core-foundations-roadmap]]). Развилки ниже помечены ❓ — решения за дизайнером._

## Зачем (фантазия и место в проекте)
Card Fusion — **скрещивание двух карт в одну** («Глитч-карта»: эффекты складываются,
стоимость усредняется). Крупнейший отсутствующий фундамент под передел классов:
- **Химик Дефектов / Глитч-Мастер** висит на нём ЦЕЛИКОМ (класс = игра вокруг фьюжна).
- **Кузнец-Рельсотрон** (сжечь карты → одна ультра) и **тег-мутации** — тоже сюда.
- Вписывается в хребет «ты — процесс, редактирующий игру» ([[paradox-mode-and-rulestack]]):
  фьюжн = авторская правка контента колоды на лету.

Class-agnostic МЕХАНИЗМ (как позиционка/RuleStack): строим примитив слияния, классовую
специфику (бесплатный фьюжн Химика, Рельсотрон Кузнеца) вешаем поверх кирпичами.

## Что уже готово (на чём строим)
- **`Card`** (`core/cards/base.py:207`): `effects` — список кирпичей; `apply` их итерирует.
  Слияние = операция над двумя `effects`-списками + правила cost/name/rarity/type.
- **Кирпичи-эффекты** самодостаточны (`execute(player,enemy,cm,is_upgraded)`) →
  конкатенация списков «просто работает» без спец-кода на эффект.
- **Ковка**: состояние по `card._fuid` → `player.deck_forge_state[uid]={level,slots:[tags]}`
  (`core/forge.py:112`). Фьюжн ОБЯЗАН определить судьбу uid/level/tags источников (❓ниже).
- **TriggerGuard / ReactionOrder** (готовы): каскад слитых эффектов конечен и упорядочен.
- **DeckManager**: hand/draw/discard/exile — фьюжн мутирует руку/колоду здесь.
- **Сим-нативность опт-ин** (паттерн positioning/debt/stakes): флаг + бот-политика →
  baseline нейтрален, дуал-сим меряет сломанный фьюжн-рулсет.

## Чего нет (надо построить)
- Чистый модуль слияния `core/fusion.py` (логика fuse, без pygame).
- UI-взаимодействие: **drag-drop в руке ОТСУТСТВУЕТ** (рука на кликах,
  `InputHandler.py:81`). Нужна новая модель ввода (❓ click-select-then-target vs drag).
- Экономика фьюжна (❓ бесплатно / энергия / мета-валюта «Глитч-заряд»).
- Бот-политика FusionPolicy (сим).

## Развилки (РЕШЕНИЯ ЗА ЮЗЕРОМ) ❓
1. **Где сливаем?** (A) В БОЮ, в руке (фантазия Химика — реактивно, в моменте) ·
   (B) у костра, как ковка (спокойно, между боями) · (C) оба. → Концепт тянет на (A).
2. **Состав Глитч-карты:**
   - **Эффекты:** конкатенация обоих списков (просто, «стопка эффектов») vs умное
     слияние однотипных (два DamageEffect → один бо́льший). → Рекомендую КОНКАТЕНАЦИЮ
     (нет спец-кода на тип, эмерджентные комбо).
   - **Стоимость:** сумма / среднее (округл.) / max / сумма−1. → Концепт: «средняя».
   - **Имя/описание:** авто («Глитч: A×B») или генерёж по эффектам.
   - **Тип/редкость:** max(rarity), тип — гибрид/«glitch».
3. **Судьба ковки источников (стык с forge):** Глитч-карта получает СВЕЖИЙ `_fuid` →
   (A) теряет level/теги обоих (просто, но карает за фьюжн прокачанного) · (B) наследует
   max(level) + объединение слотов тегов (с капом) · (C) сумма. → Рекомендую (B) с капом.
4. **Персистентность результата:** (A) Глитч живёт только до конца боя (транзиент,
   низкий риск) · (B) перманентно в колоде забега (прореживание + спайк, выше ставки). →
   Концепт «аннигиляция» намекает на (B); для среза безопаснее начать с (A).
5. **Экономика/цена фьюжна:** бесплатно (раз/ход?) · энергия · мета-валюта. → Цена нужна
   как гард-рейл (иначе вырождение); кандидат — будущая «Энтропия/Глитч-заряд».
6. **UI-ввод:** click-выбор источника → click-цель (зеркало таргетинга, реюз
   существующего паттерна) vs новый drag-движок. → Рекомендую click-click (drag-инфры нет).

## Гард-рейлы (инженерная зона — моя)
- **Кап числа эффектов** на карте (анти-бесконечная стопка → переполнение чисел/UI).
- **Пол стоимости** (фьюжн не делает 0-cost бомбу бесплатно) + кап фьюжнов за бой/ход.
- **Каскад** уже прикрыт TriggerGuard (слитые эффекты с Эхо/детонациями конечны).
- **Сим-натив:** флаг `fusion=` в `run_single_run` (дефолт off → baseline зелёный);
  FusionPolicy у бота → сломанный рулсет измерим (самоизменяемое вручную не сбалансировать).

## Предлагаемый минимальный вертикальный срез (как позиционку/RuleStack)
Доказать архитектуру ДЁШЕВО, без UI/экономики/класса:
1. **`core/fusion.py`** — чистый `fuse_cards(card_a, card_b) -> Card` (конкатенация
   эффектов + правило cost/name/rarity; политика тегов из развилки №3). + тесты. Инертно
   (никто не зовёт) → baseline зелёный.
2. **Минимальный триггер** — одна точка вызова (напр. служебная карта-«катализатор»
   фьюжна или дебаг-хук), НЕ полный UI. Доказать, что слитая карта корректно играется
   (apply итерирует объединённые эффекты, ковка-теги применяются).
3. **Сим-натив** — `fusion=` opt-in + примитивная FusionPolicy; A/B-замер; baseline off.
4. ПОТОМ (отдельными вехами): UI-ввод · экономика/Глитч-заряд · класс Химик · Рельсотрон.
