# Card Fusion — дизайн-док (v0.2, КОНЦЕПТ УТВЕРЖДЁН, реализуем ядро)

_Статус (С49): высокоуровневый концепт + развилки ядра ЗАКРЫТЫ с юзером (см. «✅ Решения»).
Реализуется поэтапно (этап 1 = чистый `core/fusion.py`). Детали способностей Химика —
позже (контент). §2 дорожной карты ядра [[core-foundations-roadmap]]._

## ✅ Решения (С49)
- **Фьюжн = механика ОДНОГО класса (Химик), НЕ всей игры.** Принцип: **МЕХАНИЗМ
  универсален и ИНЕРТЕН** (`core/fusion.py` — существует всегда, никто не зовёт), **ДОСТУП
  гейтит класс** (как `positioning_enabled`). Для 95% игры/кода фьюжна нет → баланс
  контейнеризован (сим меряет только Химика, opt-in).
- **Стоимость Глитча = `max(a, b)` с полом 1.** НЕ среднее, НЕ max+1 (max+1 при
  энергобюджете ~3 даёт неиграбельные 4-cost кирпичи). Всегда играбельно + дофамин
  эффективности + пол убивает 0-абуз. **ТОРМОЗ фьюжна — РЕСУРС класса (Реагент), не цена
  розыгрыша** ([[design-north-star]]: тормоз = валюта, не наказание).
- **Состав Глитча:** эффекты — КОНКАТЕНАЦИЯ; редкость — выше из двух (Rarity = Enum строк,
  НЕ упорядочен → нужен явный порядок тиров); тип — общий/«attack»; прокачка СБРОШЕНА
  (`upgraded=False`); флаг `is_fused`.
- **Глитч-карты ТРАНЗИЕНТНЫ на бой** (варишь заново каждый бой; оригиналы возвращаются) →
  снимает миграцию тегов ковки (разв.№3), чище и проще балансить. Перманентная ковка-
  слияние — будущий Кузнец, не Химик.

## Концепт Химика (утверждён высокоуровнево; детали способностей — потом)
«Варишь карты, не играешь». Низкая база, на лету сплавляешь карты руки в Глитч-карты,
которых нет в пулах (дофамин = авторство над контентом, [[design-north-star]]). 4 столпа:
- **Ресурс «Реагент»** (1–2/ход, тратится на слияние) — ТОРМОЗ / главная баланс-ручка.
- **Пассив «Нестабильность»** — +1 за каждый фьюжн за бой; слитые карты получают
  +Нестабильность к каждому числу. Движок кат.4 ВНУТРИбоевой (сброс между боями, как
  Мастерство/Вирулентность). Поздний бой → фьюжны чудовищны = потолок класса.
- **Способность «Сингулярность»** (раз/бой) — слить ВСЮ руку в одну карту.
- **Модульная стартовая колода** — простое сырьё под слияние.
RNG не с руки (выбор слияния детерминирован игроком — скилл-инвариант).

## Зачем (фантазия и место в проекте)
Card Fusion — **скрещивание двух карт в одну** («Глитч-карта»: эффекты КОНКАТЕНИРУЮТСЯ,
стоимость = `max(a,b)` пол 1). Крупнейший отсутствующий фундамент под передел классов:
- **Химик Дефектов / Глитч-Мастер** висит на нём ЦЕЛИКОМ (класс = игра вокруг фьюжна).
- **Кузнец-Рельсотрон** (сжечь карты → одна ультра) и **тег-мутации** — тоже сюда.
- Вписывается в хребет «ты — процесс, редактирующий игру» ([[paradox-mode-and-rulestack]]):
  фьюжн = авторская правка контента колоды на лету.

Class-agnostic МЕХАНИЗМ (как позиционка/RuleStack): строим примитив слияния, классовую
специфику (бесплатный фьюжн Химика, Рельсотрон Кузнеца) вешаем поверх кирпичами.

## Что уже готово (на чём строим)
- **`Card`** (`core/cards/base.py:207`): `effects` — список кирпичей; `apply` их итерирует.
  Слияние = операция над двумя `effects`-списками + правила cost/name/rarity/type.
- **Кирпичи-эффекты** самодостаточны (`execute(player,enemy,cm,is_upgraded)`) →
  конкатенация списков «просто работает» без спец-кода на эффект.
- **Ковка**: состояние по `card._fuid` → `player.deck_forge_state[uid]={level,slots:[tags]}`
  (`core/forge.py:112`). Фьюжн ОБЯЗАН определить судьбу uid/level/tags источников (❓ниже).
- **TriggerGuard / ReactionOrder** (готовы): каскад слитых эффектов конечен и упорядочен.
- **DeckManager**: hand/draw/discard/exile — фьюжн мутирует руку/колоду здесь.
- **Сим-нативность опт-ин** (паттерн positioning/debt/stakes): флаг + бот-политика →
  baseline нейтрален, дуал-сим меряет сломанный фьюжн-рулсет.

## Чего нет (надо построить)
- Чистый модуль слияния `core/fusion.py` (логика fuse, без pygame).
- UI-взаимодействие: **drag-drop в руке ОТСУТСТВУЕТ** (рука на кликах,
  `InputHandler.py:81`). Нужна новая модель ввода (❓ click-select-then-target vs drag).
- Экономика фьюжна (❓ бесплатно / энергия / мета-валюта «Глитч-заряд»).
- Бот-политика FusionPolicy (сим).

## Развилки
**Закрыты (С49):** №1 где сливаем = В БОЮ в руке (Химик) · №2 состав = конкатенация
эффектов + cost `max` пол 1 + редкость выше из двух + тип общий/«attack» + прокачка
сброшена · №3 судьба ковки = СНЯТА транзиентностью (Глитч живёт бой, тегам не нужна
перм-миграция) · №4 персистентность = ТРАНЗИЕНТ-на-бой. Подробности — в «✅ Решения».

**Открыты (этап 2+, контент Химика — решим по мере поступления):**
- №5 **Экономика фьюжна** — генерация/трата Реагента (фикс/ход? за карты? за урон?);
  имя ресурса. Это ГЛАВНЫЙ баланс-рычаг класса.
- №6 **UI-ввод** — click-выбор источника → click-цель (реюз таргетинга; drag-инфры нет).
- Пик-способность — «Сингулярность» (вся рука→одна) vs «Перегонка» (без расхода оригиналов).

## Гард-рейлы (инженерная зона — моя)
- **Кап числа эффектов** на карте (анти-бесконечная стопка → переполнение чисел/UI).
- **Пол стоимости** (фьюжн не делает 0-cost бомбу бесплатно) + кап фьюжнов за бой/ход.
- **Каскад** уже прикрыт TriggerGuard (слитые эффекты с Эхо/детонациями конечны).
- **Сим-натив:** флаг `fusion=` в `run_single_run` (дефолт off → baseline зелёный);
  FusionPolicy у бота → сломанный рулсет измерим (самоизменяемое вручную не сбалансировать).

## Предлагаемый минимальный вертикальный срез (как позиционку/RuleStack)
Доказать архитектуру ДЁШЕВО, без UI/экономики/класса:
1. **`core/fusion.py`** (ЭТАП 1, делаем сейчас) — чистый `fuse_cards(a,b) -> Card`
   (конкатенация эффектов + `fused_cost` max-пол-1 + редкость/тип/флаг is_fused). Без
   pygame/player/combat → тестируется без SDL. Инертно (никто не зовёт) → baseline зелёный.
2. **Класс Химик** — Реагент + движок Нестабильность + гейт «слить в руке» (флаг, как
   `positioning_enabled`). Триггер слияния руки.
3. **UI** — взаимодействие слияния всплывает ТОЛЬКО у Химика (click→click).
4. **Сим-натив** — ChemistPolicy/`fusion=` opt-in; baseline остальных зелёный.
5. ПОТОМ (вехами): пик-способность · полировка · (позже Кузнец-Рельсотрон на том же ядре).
