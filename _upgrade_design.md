# Дизайн прокачки карт = универсальный движок кат.4 (компаунд `k^f`)

> Спецификация ПЕРЕД кодом (Шаг 0). Источник правды для сим-итерации и живой
> игры. Парадигма — [[balance-curve-framework]]: враг растёт `E₀·g^f`, игрок
> аддитивно; потолок пробивают компаундящие движки кат.4, которых дефицит
> ([[scaling-audit]]). Эта система закрывает дефицит РАЗОМ для всех классов.
>
> Статус: ОБСУЖДЕНО И УТВЕРЖДЕНО с юзером. НЕ реализовано. Порядок работ — §8.

---

## 1. Зачем (место в математике)

Сила карты ≈ `base · (линейка уровней) · Π(множители тегов)`.

- **Линейка уровней (+1 за уровень)** — слой СТЕНЫ (кат.1, линейна, ОСОЗНАННО
  проигрывает `g^f`). Делает раннюю карту чуть толще, не ломает кривую.
- **Теги в майлстоунах (5/10/15)** — слой ПОТОЛКА (кат.4, компаунд).

Почему майлстоуны = `k^f` (доминирующий источник экспоненты игрока):
```
уровень ≈ c·f         (c = скорость ковки с костра — ГЛАВНАЯ ручка)
#майлстоунов ≈ уровень/5
сила ×mult ≈ k^(#майлстоунов) = k^(c·f/5) = (k^(c/5))^f
```
Условие пробития потолка из framework: `Σlog k > log(g/p)`. Конкретные ручки —
размер `k` тега, скорость `c`, шаг майлстоунов, рост врага `g`. Все собраны в §6.

### КЛЮЧЕВОЙ инсайт: условие САМО разделяет стену и потолок
Тег — это `k × P(условие выполнено)`. В **wall-метрике** (бот не строит билд)
`P→0` → эффективный множитель ≈1 → **стена остаётся честной**. В **ceiling**
(билд заточен под условие) `P→1` → полный `k` → **потолок пробивается**.
⇒ Условный компаунд самоограничен в стене → СОХРАНЯЕТ зазор стена↔потолок
(всё пространство геймплея). Безусловный ×k надувал бы и стену → гонка
вооружений с врагом → зазор схлопывается. Поэтому теги > чистые мультипликаторы.

Это поглощает план [[wall-ceiling-taxonomy-plan]]: не нужно вручную клеить
статусам ярлык wall/ceiling — условный тег ПО ПОСТРОЕНИЮ ceiling-инструмент,
а до выполнения условия лежит как wall-флат.

## 2. Архитектура паспортов карт (решение юзера: тег на ЭКЗЕМПЛЯРЕ)

Тег живёт на КОНКРЕТНОЙ карте (этот «Удар» прокачан до 15 с ×mult «из пустой
руки»; другой «Удар» в той же колоде — голый). Это даёт «экспоненту
реиграбельности»: комбинация разных грейдов одной карты в одной колоде.

Чтобы НЕ раздувать объект карты и сохранить плоскую (sim-friendly) архитектуру —
**разделяем сущности**:

- **Объект `Card`** несёт только идентичность: `card_id` (тип, напр. `"strike"`)
  + `uid` (уникальный инстанс в этом забеге). Логика розыгрыша — без изменений.
- **Вся мета-прокачка** — в ЕДИНОМ плоском словаре игрока
  `player.deck_forge_state: dict[uid → ForgeRecord]`.

```
ForgeRecord = {
    "level": int,                 # 0..∞ (линейка +1)
    "slots": [TagInstance, ...],  # открытые майлстоунами слоты
}
TagInstance = { "tag_id": str, "kind": "add"|"mult", "value": float }
```

Почему так:
- **Sim:** headless-расчёт = быстрый поиск по словарю (`deck_forge_state[uid]`),
  не нужно таскать состояние внутри боевого объекта.
- **UI:** карта просто обрастает СПИСКОМ условий в описании (чистый рендер,
  растёт вниз). Бейдж уровня = `level`.
- **Персистентность:** `deck_forge_state` живёт весь забег, НЕ сбрасывается
  `reset_combat_statuses` (как `persistent_allies`). Реюз стартует на новом забеге.

### Врезка в существующий код
- `Card.__init__`: добавить `card_id` + `uid` (генерится при создании инстанса;
  в симе — детерминированный счётчик, БЕЗ `Math.random`/`uuid`-недетерминизма).
- ВНИМАНИЕ: сейчас апгрейд = `self.name += "+"` + bool `self.upgraded`. Старый
  одноступенчатый upgrade МИГРИРУЕТ в `level≥1` (level 1 = текущий `+`). Старые
  карты с `upgraded=True` читаются как `level=1` (обратная совместимость).
- Канал «какая карта играется» — переиспользовать `_card_being_played`
  (CombatManager:141/157, уже есть для Потока). Tag-движок берёт `uid` оттуда.

## 3. Экономика ковки (источник очка `c` + босс-капы)

### Базовый источник — КОСТЁР (ручка `c`)
Костёр (`ui/Campfire.py`, этаж `FLOORS_PER_ACT-2` опц. с магазином) получает
4-е действие **«Ковка»**: потратить ресурс → +1 уровень выбранной карты.
Скудость ресурса = стратегия «что качать» (концентрировать в 2-3 карты =
острый билд, размазать = универсал). Экономика = РЕГУЛЯТОР гонки (framework §6).

- **Ресурс ковки — отдельная валюта `FP` (Forge Points), УТВЕРЖДЕНО.** НЕ золото:
  чище экономика (ковка не конкурирует с магазином/реликвиями за один кошелёк),
  и `FP` лимитируется ЭТАЖАМИ → прямой контроль скорости `c` (= наклона игрока)
  независимо от золотого дохода. `player.forge_points`. Капля FP за бой/этаж
  (точный дроп — калибровка), плюс возможный бонус с босса.
- **Стоимость уровня — РАСТУЩАЯ ВНУТРИ ТИРА, УТВЕРЖДЕНО.** Цена +1 уровня растёт
  с текущим уровнем карты в пределах тира (напр. `FP_cost = base + level·step`),
  сбрасывает «крутизну» на новом тире после босс-гейта. Эффект: тактический
  прессинг — добить карту до майлстоуна дорого → выбор «дожать одну до ×mult или
  поднять две до +mult». Естественный анти-degenerate тормоз концентрации.
- Накопление в БОЮ — ВЫРЕЗАНО из базового движка (защита сима от петель
  «играй→качай→играй» и ранних рантануть). Уходит в пул РЕДКИХ артефактов позже.

### Контрольные точки — БОСС-ГЕЙТЫ (снимают КАП уровня, не дают очков)
Босс открывает следующий ярус прокачки: до него уровень карты заперт капом.
Это вторая, НЕЗАВИСИМАЯ ручка стены: даже при быстрой ковке (высокий `c`) игрок
не превратит уровни в `×mult`, пока не убил нужного босса. ⇒ кусочно-экспонента
игрока: медленный старт (только `+mult` разрешён) → разблокировка `×mult` после
босса. Контроль взрыва чисел ВСТРОЕН в прогрессию, а не только в формулу.

**Бонус:** это и есть [[boss-filter-ladder]] (боссы-фильтры, к которым сим был
слеп) — но встроенный в прогрессию, которую сим БУДЕТ моделировать. Босс
перестаёт быть невидимой стеной, становится видимой контрольной точкой компаунда.

### ✅ Развязка двух шкал (УТВЕРЖДЕНО)
Две РАЗНЫЕ оси, увязаны жёстко:
- **Уровни КАРТЫ** с майлстоунами: **5 / 10 / 15** (шкала прокачки).
- **Босс-ЭТАЖИ** в коде: каждые **20** (`FLOORS_PER_ACT`) → 20/40/60/80/100.

| Босс-этаж | Снимает кап уровня карты до | Открывает |
|---|---|---|
| 20 | **5** | слот-1 (ранний тег, `+mult`) |
| 40 | **10** | слот-2 (средний тег, `+mult`) |
| 60 | **15** | слот-3 (**легендарный**, `×mult`) |
| 80 / 100 | 20 / 25 (расширение) | слот-4+ (позже) |

⇒ `×mult` физически недостижим раньше этажа ~60 → **железная защита ранней
стены** (сим знает потолок компаунда на каждом этаже). Числовые значения капов —
финально калибруются в симе, но СТРУКТУРА (босс↔майлстоун) зафиксирована.

## 4. Слоты и теги (мягкий старт → истинный компаунд)

Майлстоун уровня открывает СЛОТ под тег. Теги расслоены по силе:

| Майлстоун (уровень карты) | Слот | Класс тега | Эффект |
|---|---|---|---|
| 5  | слот 1 | ранний | `+mult` (аддитивный, мягкий) |
| 10 | слот 2 | средний | `+mult` (аддитивный) |
| 15 | слот 3 | **легендарный** | **`×mult`** (мультипликативный, ИСТИННЫЙ компаунд) |
| 20/25/… | слот 4+ | (расширение) | — позже |

- Несколько тегов на ОДНОЙ карте стакаются → произведение при выполнении
  нескольких условий (максимум билд-комбинаторики).
- **Получение тега — ДРАФТ ИЗ 3, УТВЕРЖДЕНО.** При открытии слота игрок выбирает
  1 из 3 случайных тегов, соответствующих ТЕКУЩЕМУ тиру майлстоуна (слот-3 → 3
  легендарных `×mult`-варианта, не ранние). Реиграбельность + осмысленный выбор
  билда. Пул-кандидатов фильтруется Smart Weighting (§10.1) — родственные
  текущему билду теги всплывают чаще. Для сима: драфт = детерминированный выбор
  ceiling-ботом лучшего из 3 под архетип (см. §8); wall — случайный/без ковки.
- **Инвариант для сима:** `×mult`-тег ФИЗИЧЕСКИ не существует до уровня 15, а
  уровень 15 недостижим до босса-60 (§3). ⇒ сим ЗНАЕТ: компаунд `×mult` не
  появляется раньше этажа ~60 → ранняя стена математически защищена.
- UI: карта показывает список «[уровень] эффект · теги: …». Чистый рендер.

## 5. Реестр предикатов-условий (стартовый пул)

Data-driven, как `DetonationRegistry`: запись = `{tag_id, predicate(ctx), kind,
value, label}`. Новое условие = 1 запись + предикат. `ctx` = срез боя в момент
удара (рука, поле, цель, счётчики хода).

### Generic (нейтральные, в общем пуле)
| tag_id | Условие (предикат) | Резонанс с классом |
|---|---|---|
| `empty_hand` | рука пуста на момент розыгрыша | темпо/Разбойник (быстрый слив руки) |
| `first_card` | 1-я карта за ход | контроль, заряд-карты |
| `nth_card` | N-я+ карта за ход (комбо-цепь) | Маг (комбо), цепные ходы |
| `low_hp` | HP игрока < порога | Берсерк (HP-как-ресурс) |
| `full_combo` | у цели ≥K разных статусов | Друид/стихийники |

### Классовые (резонанс с движком кат.4 — идентичность без класс-кода)
| tag_id | Условие | Класс |
|---|---|---|
| `per_minion` | за каждого живого союзника | **Призыватель** (Свора, N²) |
| `target_poisoned` | у цели есть Яд (масштаб от стака) | **Друид** (Вирулентность) |
| `shielded` | у игрока есть щит/Барьер | **Воин** (Барьер = атака) |
| `per_bleed` | за стак Кровотечения на цели | **Разбойник** (Кровожадность) |
| `missing_hp` | за % недостающего HP | **Берсерк** (Ярость крови) |
| `per_combo_done` | за каждое комбо в этом бою | **Маг** (Мастерство) |

Резонанс намеренный: одна система прокачки выражается по-разному на каждом
классе → идентичность бесплатно. Теги перекликаются с уже построенными
движками (Вирулентность/Барьер/Мастерство/Свора/Кровожадность).

## 6. Формулы-ручки (предмет калибровки в симе)

Все числа НИЖЕ — стартовые гипотезы, КАЛИБРУЮТСЯ симом против `g`. Реальные —
после §8. Заносятся в `_balance_knobs.md` (раздел «Прокачка») после калибровки.

| Ручка | Симв. | Гипотеза | Смысл / влияние |
|---|---|---|---|
| Скорость ковки | `c` | ~1 уровень / костёр | наклон экспоненты игрока (костров ~1/акт) |
| Бонус линейки/уровень | `δ` | +1 к base-эффекту | слой стены (кат.1) |
| Ранний тег | `+a` | +0.5 mult | мягкий старт (аддитивно к множителю) |
| Легендарный тег | `×k` | ×1.5 при условии | истинный компаунд (кат.4) |
| Шаг майлстоунов | `s` | 5 уровней | реже майлстоун = медленнее `k^f` |
| Кап до босса-N | — | 5/10/15 | ступень разблокировки (§3) |
| Рост врага (есть) | `g` | HP 1.028 / DMG 1.026 | EnemySpawner (для сравнения наклонов) |

Условие баланса (из framework): подобрать `c, k, s` так, чтобы СОБРАННЫЙ билд
(ceiling) флипал наклон в плюс (`(k^(c/s)) > g`), а НЕсобранный (wall) — нет
(`P(условие)→0` держит множитель ≈1). Зазор = доказательство двойной экспоненты.

## 8. Порядок работ (УТВЕРЖДЁН — «сначала сим видит условия»)

Урок слепоты сима ([[balance-findings-shock-dilution]], Кровожадность): движок,
который сим не пилотирует, НЕЛЬЗЯ откалибровать. Поэтому:

1. **Сим: модель FP + уровней карт** — `deck_forge_state` + `forge_points` в
   sim-игроке, ковка на костре в runner (ceiling-бот концентрирует FP в ядро
   билда по растущей цене; wall — не качает или размазывает). Босс-капы уровня.
   **Trigger Cap (§10.2) закладывается СРАЗУ** — счётчик глубины в bot/runner
   (защита headless-прогона от зависания с первого шага).
2. **Сим: предикаты в damage-calc** — ceiling-бот ВЫПОЛНЯЕТ условия (играет
   так, чтобы `empty_hand`/`per_minion`/… срабатывали); wall — нет. Иначе тег
   невидим (грабли Шока). Драфт тегов из 3 + **Smart Weighting (§10.1)** —
   ceiling-бот берёт лучший под архетип, affinity-вес моделируется.
3. **Калибровка** — свип `c, k, s, affinity` против `g`: flip наклона у ceiling,
   плоская стена у wall. **Процент-конвенция (§10.3)** даёт масштаб-инвариантные
   формулы → проще свип. Зелёный регресс-гард (`baseline.py`).
4. **Чистка слепых зон классов** — каждый класс-предикат реально пилотируется
   ботом (резонанс §5 измерим).
5. **ТОЛЬКО ПОСЛЕ зелёного теста** — раскатка в живой код: `Card.uid`,
   `ForgeRegistry`, ковка+FP в `Campfire.py`, чтение тегов в боёвке, драфт-UI,
   UI-список, босс-капы, Trigger Cap в реал-боёвке. + тесты + headless + доки.
6. Big-numbers UI — отдельная поздняя итерация (Balatro-формат). Процент-скейл
   (§10.3) — сквозная конвенция, внедряется по ходу (не отдельный этап).

## 9. Развилки (ВСЕ РЕШЕНЫ — юзер, ревью дока)

1. **Шкалы карта↔босс** — ✅ УВЯЗАНЫ ЖЁСТКО: босс-20→кап ур.5, босс-40→кап ур.10,
   босс-60→кап ур.15 (§3, таблица). Железная защита ранней стены.
2. **Ресурс ковки** — ✅ отдельная валюта **`FP` (Forge Points)**, лимит этажами
   (§3). Чистая экономика + прямой контроль `c`.
3. **Стоимость уровня** — ✅ **РАСТУЩАЯ внутри тира** (`base+level·step`, сброс
   крутизны на новом тире). Тактический прессинг, анти-degenerate (§3).
4. **Получение тегов** — ✅ **драфт из 3** вариантов текущего тира (§4), пул под
   Smart Weighting (§10.1).

## 10. Глобальные гард-рейлы под экспоненту пула (УТВЕРЖДЕНО)

Три предохранителя — закладываются В АРХИТЕКТУРУ СИМА с самого начала (§8), не
прикручиваются потом. Защищают пул и числа от вырождения при росте контента.

### 10.1 Smart Weighting (анти-размытие пула)
**Проблема:** растущий пул тегов/артефактов → драфт из 3 даёт случайный мусор не
по билду (Pool Dilution) → игрок не может СОБРАТЬ архетип → ceiling не пробивает.
**Решение:** шанс выпадения тега/артефакта повышается, если он РОДСТВЕНЕН уже
выбранным в билде (общий `tag_id`-семейство / класс-резонанс §5 / тема колоды).
Переиспользует `_deck_themes`/`_card_score` из `managers/balance/builds.py`
(там уже есть бонус за тему). Реализация: вес кандидата = `base · (1 + affinity)`,
`affinity` = совпадение с текущими тегами/архетипом.
**Для сима:** ceiling-бот и так берёт лучшее под архетип — Smart Weighting
делает это ДОСТУПНЫМ (нужные теги реально выпадают), сокращая разрыв «теория vs
что выпало». Ручка: сила `affinity` (0 = чистый рандом = размытие, ∞ = всегда
по билду = нет разнообразия). Калибруется.

### 10.2 Trigger Cap (анти-бесконечный цикл)
**Проблема:** цепочки «Эха» (ретриггер) + детонации могут рекурсить
(удар→эхо→детонация→ещё удар→…) → бесконечный цикл в реал-тайме + переполнение
чисел. Уже частично снято в детонациях (RAW `take_damage` вне `calculate_damage`,
см. трекер D1), но Эхо + теги добавят новые пути рекурсии.
**Решение:** ЖЁСТКИЙ глобальный счётчик вложенных триггеров на одно действие —
**макс 5** (`MAX_TRIGGER_DEPTH`). Достигнут → цепочка обрывается, лог. Считает
ВСЕ ретриггеры/детонации/каскады тегов суммарно за один розыгрыш карты.
**Где:** счётчик в `CombatManager` (зеркало в bot/runner), инкремент на входе
в каждый триггер-хендлер (Эхо/детонация/каскад тега), сброс по завершении
розыгрыша (рядом с `_card_being_played = None`). Ручка: `MAX_TRIGGER_DEPTH=5`
→ `_balance_knobs.md`.
**Для сима:** защищает headless-прогон от зависания + делает урон ОГРАНИЧЕННЫМ
сверху за действие (предсказуемость замера).

### 10.3 Процентный скейл (анти-инфляция плоских чисел)
**Проблема:** при big-numbers (урон в тысячах к эт.100) плоские ивенты/обмены
(«−10 HP», «+25 золота») становятся неощутимыми → мёртвый контент.
**Решение:** ВСЕ ивенты/обмены/жертвы переводятся с плоских чисел на ПРОЦЕНТЫ
от текущего ресурса (% HP / % золота / scale-by-floor). Прецедент уже есть:
костёр лечит `REST_HEAL_PCT=0.30` от недостающего HP (Сессия 38), `lose_hp`
сквозь щит. Распространяем конвенцию: Ритуал крови, Проклятый сундук,
карты-жертвы, цены — всё в %.
**Для сима:** проценты МАСШТАБ-ИНВАРИАНТНЫ → одна формула работает на эт.5 и
эт.100, не нужно ре-тюнить плоские пороги под каждый тир. Упрощает калибровку.
**Конвенция под рост (§ из [[wall-ceiling-taxonomy-plan]]):** новый ивент
ОБЯЗАН быть в %, не во флате. Аналог регресс-гарда — правило при добавлении.
