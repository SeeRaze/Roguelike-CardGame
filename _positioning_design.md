## 0. Зачем (кейстоун, не фича класса)
Позиционка 3×3 = СУБСТРАТ, на котором висит половина Парадокс-режима (Контрабанда /
Геометрический Коллапс / Мёртвые Души-арена) И куча классов (Авангард, позиции
саммонов). Это субстрат целого РЕЖИМА и грядущего class-redesign, а не отдельная
механика. Поэтому планируем отдельной вехой и НЕ биг-бэнгом (дисциплина §8 RuleStack).

## 1. Текущая модель боя (сверено с кодом, Jun 7)
- `CombatManager`: `player` + `enemies` (list) + `allies` (list — саммоны). Пространства НЕТ.
- Таргетинг: `get_target_enemy()` (живой враг), AoE = «все враги»; враг бьёт СЛУЧАЙНУЮ
  цель из {player + живые allies} — `Enemy._choose_attack_target` (С37, фундамент под
  «провокацию»). Саммоны Призывателя живут в `allies`, `MAX_PERSISTENT_ALLIES`.
- Карты: одиночная цель ИЛИ все враги. Никаких рядов/слотов.

## 2. РАЗВИЛКИ — решить с юзером ПЕРЕД кодом
**A. Что на сетке?**
  (a) ДВЕ стороны по 3×3 (партия игрока + враги); (b) одна общая 3×3 арена;
  (c) МИНИМУМ среза — только сторона игрока (player+саммоны) на 3 слота, враги пока «толпа».
**B. Что позиция ДАЁТ механически?** (выбрать стартовый набор)
  — фронт/тыл (ближний бой бьёт фронт, дальний — тыл); — адъяценция (AoE/сплеш по соседям);
  — защита (фронт танкует за тыл = Авангард→щит, `backlog-artifact-ideas`);
  — «толстые пилоны» (память: позиционка их ломает в Коллапсе).
**C. Геометрия среза:** полная 3×3 (9 слотов) сразу ИЛИ 1 ряд из 3 (фронт-линия) → расширить.
**D. Совместимость карт:** как одиночная/AoE цель ложится на слоты (фронт по умолчанию?
  выбор слота игроком?). Старые карты ОБЯЗАНЫ работать без правок.
**E. Враги на сетке + ИИ-таргетинг по позиции** ИЛИ враги пока без позиции в срезе.
**F. Сим-нативность:** позиционка OFF в baseline (как stakes/debt/forge, opt-in) ИЛИ сразу
  базовая? (Рекоменд: opt-in/off → baseline зелёный, бот учится позже.)

**МОЯ РЕКОМЕНДАЦИЯ (для обсуждения):** срез = A(c) + C(1 ряд из 3) + B(фронт/тыл +
Авангард-защита) + F(off в baseline). Минимальная поверхность, доказывает субстрат,
не биг-бэнг. Полная 3×3 + враги-на-сетке + адъяценция = последующие итерации.

## 3. Прерквизиты / риски
- НЕ ломать текущий бой (одиночная/AoE цель, саммоны, `_choose_attack_target`).
- Сим-нативность + baseline зелёный (паттерн stakes/debt/forge: позиционка opt-in,
  дефолт off → `_StubGM`/runner не задеты).
- UI грида — отдельная подзадача (рендер слотов + выбор цели), headless-проверка.
- **Связь с class-redesign:** позиции = субстрат новых классов → семантику слотов
  согласовать с концептами (`class-concepts-ideas`) ПЕРЕД фиксацией.

## 4. Предлагаемый план (DRAFT — уточнить после развилок; 1 коммит = подзадача)
1. ЯДРО: структура слотов (Grid/Slot — чистый модуль) + опц. позиция на существе
   (дефолт None → нет позиции, инертно). +тесты. baseline зелёный.
2. ТАРГЕТИНГ: фронт/тыл-семантика в `get_target_enemy`/AoE + `_choose_attack_target`

3. ЗАЩИТА: Авангард (фронт танкует/щит за тыл) — первая позиционная механика. +тесты.
