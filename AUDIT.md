## 1. Латентные неточности (📝)

### 1.1 Превью урона forge-тегов × мульти-хит
Ховер-тултип карты показывает множитель комбо как `×N` **один раз**, по снимку
первого удара. У мульти-хит карт каждый последующий удар может видеть другое
состояние комбо (комбо инкрементится между ударами) → превью слегка занижает.
**Почему принято:** превью — ориентир, не точный предсказатель; точная симуляция
каждого удара = заметная сложность ради косметики. Реальный удар считается верно.
Канал: `core/EffectCalculator.py` (шаг 7), снимок `CombatManager._build_play_snapshot`.

### 1.2 Снимок розыгрыша без try/finally
`CombatManager.play_card_by_index`: `_card_being_played`/`_play_snapshot` чистятся
в конце метода, не в `finally`. Если исключение внутри `selected_card.apply` —
снимок останется висеть до следующего розыгрыша.
**Почему 🟢:** теоретический край — исключение в `apply` означает сломанную карту,
бой всё равно нерабочий. Ранние return (нет энергии/цели) идут ДО установки снимка.

### 1.3 Дубль HP в бою (UI)
HP игрока показан дважды: строка ресурсов слева-сверху (`ui/resource_hud.py`) +
панель героя (`ui/combat/panels.py::draw_player_panel`, HP-бар + число).
**Статус:** 👀 не баг, визуальный шум. Оценить вживую, возможно убрать HP из строки
в состоянии COMBAT (но это ломает «единый вид строки на всех экранах»).

## 2. Приближения симулятора (📝)

Симулятор (`managers/balance/`) — инструмент тюнинга, НЕ точная копия игры.
Известные расхождения, влияющие на интерпретацию чисел:

### 2.1 Sim слеп к боссам-фильтрам
Симулятор моделит гладкую кривую врага `E₀·g^f`, но дискретные боссы-фильтры
(эт.20/40/60/80/100) не видит → их тюнинг вслепую. Форж-капы (5/10/15 по боссам)
УЖЕ работают как один из фильтров и симу видны; явные классовые ворота — нет.
→ [[boss-filter-ladder]]. См. также §3.1.

### 2.2 Бот не пилотирует часть синергий идеально
Слой `BotPolicy._synergy_pick` (`managers/balance/policy.py`) детектит детонации/
Раскол/Шок/Поток, но НЕ все билды. Награды разбавляют колоду «мусором» из generic-
пула, который бот тащит → глубокие классы в sim занижены относительно реального
скилл-пилота. Замер «честнее, не идеален». → [[balance-findings-shock-dilution]].

### 2.3 Прокачка карт: атак.теги мизерны по покрытию
Per-card forge-теги применяются ×N верно, но несут лишь ~9-16 атак/забег (ковка
с костра тегает 3-4 карты, редко вытягиваются). Потолок ФЛИПАЕТ не ими, а
«Заточкой» (`player.atk_mult` — компаунд-% на ВСЕ атаки). Per-card теги = late-game
усилитель (act3+), не основной рычаг. → [[balance-findings-dps-bound]].

## 3. Открытые механики (🔧)

### 3.1 Боссы-фильтры — НЕ реализованы (приоритет: MEDIUM)
Мягкие боссы эт.20/40/60/80/100 с обходным путём для каждого класса. Предусловия:
эт.40 завязан на экономику прореживания (✅ готова), эт.60 — на движки кат.4
у всех (✅ готовы). Симулятору нужны чекпойнты-ворота, иначе тюнинг вслепую.
Форж-капы — единственный работающий фильтр сейчас. → [[boss-filter-ladder]].

### 3.2 Сустейн-движок Друида — бэклог (приоритет: LOW)
Druid med флипает, но wr100=0 — оборонная стена держит хвост. Движок Вирулентности
АТАКУЮЩИЙ → не двигает оборонный потолок. Нужен сустейн-движок (защитный компаунд).
→ [[balance-findings-druid-engine]].

### 3.3 Эхо × Детонация — правило согласовано, не реализовано
Эху разрешено дублировать детонацию (свинг за комбо) с transient frame-guard
против петли. Технический крючок: Эхо повторяет РАЗРЕШЁННЫЙ бурст (урон-only),
а не пере-триггерит ворота `requires()` (статусы уже сняты). → [[boss-filter-ladder]].

## 5. Решено (✅, для истории)

- **Прокачка карт флипает потолок** — «Заточка» (DPS) + «Закалка» (HP) как два стока
  FP; тема-гейт колоды офенс/оборона. War 56→97, Rogue 26→72, Mage 62→101 и т.д.
  → [[balance-findings-dps-bound]], [[economy-trinity-survival-engine]].
- **Движки кат.4 у всех 6 классов** — Барьер/Мастерство/Эхо/Кровожадность/
  Вирулентность/Свора. Framework `Σlog k > log(g/p)`. → [[balance-curve-framework]].
- **Единый расчёт урона (превью == удар)** — `EffectCalculator` развязан `dry_run`
  (гасит только побочки, множители всегда). Костыль `atk_mult` удалён. Аудит C40.
- **Druid engine** — движок работает и бот пилотирует, но зазор +4: стена
  оборонительная, движок атакующий. Принято. → [[balance-findings-druid-engine]].
- **Прореживание колоды (экономика)** — почти нейтральный рычаг в симе; реальный
  регулятор сборки = драфт реликвий/карт. → [[balance-findings-economy-thinning]].
- **Стихии + комбо** — Шок/Земля/Воздух + 6 меж-стихийных комбо (5 детонаций +
  распространение). Два реестра: ComboRegistry (×N) + DetonationRegistry (бурст).
